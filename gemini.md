# gemini.md

Purpose: repository guidance for Gemini CLI experiments in this workshop.

When working in this repository:
- Optimize for beginner-friendly outputs that learners can execute in one sitting.
- Prefer markdown tables, short checklists, and JSON shapes over long prose.
- Separate learner-facing guidance from instructor-facing guidance when both exist.
- Rank recommendations by impact and effort.
- Cite the actual workshop files used before making recommendations.
- Avoid assuming network access or live API availability when a mock-first path exists.
- For web app examples, keep secrets server-side and call out error handling clearly.

Testing note:
- If your Gemini CLI setup does not automatically apply this file, include `@gemini.md` explicitly in the prompt so you can still compare the result.
